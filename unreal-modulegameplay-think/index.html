Unreal 특정 게임에 적용할 모듈형 개발 구조 고민

2025년 10월 9일
#UnrealEngine #GameFeature

개요

Lyra 를 보며 GameFeature 의 활용 방법이 정말 다양한 것 같다는 생각이 들었다. 그 방법은 어떤 게임

을 만들 것인지에 따라 결정될 것으로 보였다. Lyra 는 슈팅 게임에 최적화된 GameFeature 들로 구성

되었다. 본인은 RPG , 액션 게임 장르를 좋아하기 때문에 이에 맞는 구조를 설계해보려고 한다.


어떤 게임에 적용할 것인가?

  • 게임 정의 : 똑같은 룰 안에서 캐릭터와 스킬들이 달라질 수 있는 게임

게임에 맞는 구조를 설계해보면

  • 게임의 룰은 GameMode 로 구현
    • 라이라의 Experience 를 사용하지 않고, 기존의 GameMode 를 사용
  • 게임의 캐릭터 변경은 Lyra 의 Experience → CharacterDefinition 으로 역할을 바꿔 구현
    • 기존 Experience 가 가진 PawnData, AbilitySet 등은 그대로 사용
  • 게임 피처는 크게 3가지로 나눠 개발
    • 캐릭터 콘텐츠 : 캐릭터들이 할 수 있는 코어 액션들을 하나하나 정의
      • 라이라의 GameFeature 는 현재 게임의 코어 액션을 정의
        • ex) ShooterCore - 인벤토리 시스템 등
        • ex) TopdownArena - 카메라 시스템 등
      • 캐릭터가 할 수 있는 액션들을 GameFeature 로 분리
        • GameFeature_Skill_Stinger // 돌진 공격
        • GameFeature_Skill_Fireball // 화염구
        • GameFeature_Skill_Teleport // 순간이동
    • 적 콘텐츠 : 새로운 적 추가
    • 맵 콘텐츠 : 코인, 새로운 영역, 새로운 던전, NPC 등

GameFeature 주의사항

  • 게임 피처들은 코어 게임을 전혀 모르도록 해야 함.
    • 즉 의존성이 없도록 하여 활성화 비활성화가 자유롭게 구현
  • 커스텀 게임 피처들을 제작해 재사용할 것
    • 제공된 게임 피처들 말고도 다양하게 커스텀해서 생산성을 올릴 수 있음.
  • 게임 피처 관련 리소스는 해당 피처 플러그인 폴더에 넣을 것
    • 완전한 모듈형 개발을 위해 분리
  • 데이터 관련 피처 액션을 활용할 것
    • 에셋 매니저를 활용해서 상점과 같은 콘텐츠에서 자동으로 등록되도록 함.
  • 멀티 플레이 환경에서 게임 피처가 활성화되면, 모든 클라이언트에게 적용하기 때문에 특정 클라이언트에게만 적용하도록 하는 게임 피처 필터가 필요.
